## 一、第一輪（水平測試）考試技巧

### 時間管理
- 總時長 120 分鐘，滿分 100 分
- 建議分配：單選題 30 分鐘 → Python 編程 40 分鐘 → AI 問題解決 50 分鐘
- 先做有把握的題目，標記不確定的題目最後回頭做

### 單選題技巧
- AI 概念題：注意區分相似概念（如 precision vs recall、過擬合 vs 欠擬合）
- 數學題：線性代數和概率是高頻考點
- 算法題：時間複雜度分析是常見陷阱
- **排除法**：不確定時先排除明顯錯誤選項

### Python 編程題技巧
- 注意邊界條件和異常處理
- 代碼要簡潔可讀，變量命名清晰
- 熟悉 NumPy 向量化操作（比循環快）
- 常見考點：列表推導式、字典操作、文件讀寫、正則表達式

### AI 問題解決技巧
- 理解題意比直接寫代碼更重要
- 先畫出數據流程圖再動手
- 注意數據預處理步驟（缺失值、標準化、編碼）
- 模型選擇要根據問題類型（分類/回歸/聚類）

## 二、第二輪（中國站）上機技巧

### 玻爾平台使用
- 提前熟悉 Bohrium 平台的操作界面
- 了解如何調用雲端 GPU 資源
- 熟悉 Jupyter Notebook 環境
- 注意平台的文件上傳和下載限制

### AB 數據集打榜策略
- **A 數據集**：通常是公開的，可以用來調試和優化
- **B 數據集**：通常是隱藏的測試集，只在最後提交
- 策略：在 A 數據集上充分調優，但**不要過擬合 A 數據集**
- 使用交叉驗證估計模型在 B 數據集上的表現
- 提交前確保代碼可以獨立運行（不要依賴特定的臨時文件）

### 模型訓練技巧
1. **先建立 baseline**：用簡單模型（如 Logistic Regression）先跑一遍
2. **逐步優化**：baseline → 特徵工程 → 複雜模型 → 集成
3. **記錄實驗**：每次實驗的參數和結果都要記錄
4. **學習率**：從 0.001 開始嘗試，使用 learning rate scheduler
5. **正則化**：Dropout、L2 正則化、Early Stopping
6. **數據增強**：如果數據量不足，考慮數據增強技術

### 常見踩坑
- 數據洩露：在預處理時使用了測試集的信息
- 類別不平衡：忘記處理不平衡的標籤
- 内存溢出：批量加載數據時沒有控制 batch size
- 隨機種子：忘記設置隨機種子導致結果不可復現

## 三、通用學習建議

### 必備技能清單

#### 基礎層（第一輪必備）
- [ ] Python 基礎語法（變量、循環、條件、函數、類）
- [ ] Python 數據結構（列表、字典、集合、元組）
- [ ] NumPy 基礎操作
- [ ] Pandas 數據處理
- [ ] Matplotlib / Seaborn 數據可視化
- [ ] 線性代數基礎
- [ ] 概率論與統計基礎
- [ ] 微積分基礎（梯度、鏈式法則）
- [ ] 監督學習基本概念
- [ ] 常見 ML 算法原理

#### 進階層（第二輪必備）
- [ ] PyTorch 基礎（Tensor、Autograd、nn.Module）
- [ ] CNN 原理與實現
- [ ] RNN / LSTM 原理與實現
- [ ] Transformer / Attention 機制
- [ ] 模型訓練全流程（數據加載、訓練循環、驗證、測試）
- [ ] 超參數調優技巧
- [ ] 模型部署基礎
- [ ] Kaggle 競賽經驗

#### 拔高層（衝刺 IOAI）
- [ ] 預訓練模型微調（BERT、GPT）
- [ ] 計算機視覺 SOTA 方法
- [ ] NLP SOTA 方法
- [ ] 閱讀英文論文能力
- [ ] 多 GPU 訓練
- [ ] 模型壓縮與加速

### 推薦練習平台
1. **Kaggle**：參加入門競賽（Titanic、House Prices）
2. **LeetCode**：算法和數據結構練習
3. **玻爾平台**：熟悉官方競賽環境
4. **Google Colab**：免費 GPU 練習 PyTorch

### 推薦學習資源
1. **吳恩達 Machine Learning Specialization**（Coursera）
2. **fast.ai Practical Deep Learning**（免費）
3. **PyTorch 官方教程**（pytorch.org/tutorials）
4. **《動手學深度學習》**（d2l.ai，有中英文版）
5. **3Blue1Brown**：線性代數和神經網絡的直覺理解
